Почему так

Инженерные решения под реальный посёлок

📶 Offline-first

UUIDv7 на клиенте, IndexedDB-очередь, Background Sync. POST идемпотентен — повтор не плодит дублей.

⚡ Live без Redis

SSE поверх Postgres LISTEN/NOTIFY. Один Node-процесс, без отдельного WebSocket-сервера.

🤖 Telegram-логин

Регистрация и вход через бота, модерация командиром. Без паролей — фактор владения аккаунтом.

🔒 PII-минимум

Обязателен только адрес дома. Список домов и фамилий — вне репозитория. ФЗ-152.

📷 Прямо в S3

Фото жмётся в WebP на клиенте и грузится по pre-signed PUT, минуя бэкенд. EXIF стрипается.

🧾 Аудит-журнал

Append-only incident_events — источник timeline в UI и будущих отчётов.

Запуск

Telegram-бот как полный интерфейс жителя

Уведомления + создание инцидента прямо в боте (текст/фото/гео/голос). Покрывает не-PWA половину посёлка — устройства ~50/50, много кнопочных.

Запуск

Реальный iOS-push и онбординг

A2HS + web-push на живых iPhone по чеклисту, регистрация «бабушка справится», операционный минимум (health-check, бэкапы, логи).

Качество жизни

Замы командира · голос/видео · экспорт

Снять единую точку отказа, push-эскалация по таймауту, медиа в комментариях, экспорт PDF/CSV из журнала событий.

Горизонт

Mesh между жителями

Peer-sync (Yjs) при полном отсутствии интернета. UUIDv7 + append-only events совместимы. Плюс i18n и цифровые подписи (TSP).
