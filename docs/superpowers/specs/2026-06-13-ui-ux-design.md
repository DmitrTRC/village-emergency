# village-emrg UI/UX — Design Spec

**Дата:** 2026-06-13
**Статус:** утверждён к планированию
**Связано:** `docs/superpowers/specs/2026-06-11-village-emrg-design.md` (доменный спек)

## Проблема

Фронтенд функционально готов (Plan 2), но не имеет UI: ноль CSS, голый
семантический HTML, и — главное — **экран создания инцидента `/new`
недостижим из интерфейса** (на него ничего не ведёт). Приложение отталкивает, а
должно притягивать: это система оповещения для всего села (от подростков до
пожилых) плюс командир ДНД. Ключевое требование — мгновенная понятность под
стрессом и «красная кнопка», в которую невозможно промахнуться в перчатках и на
морозе.

## Зафиксированные решения (из брейншторма)

- **Эстетика:** тёмная «тревожная кнопка» — высокий контраст, цвета уровня
  считываются мгновенно. Тёмная тема уже заявлена (`theme-color #0b1220`).
- **Навигация:** нижний таб-бар **Лента · Карта · Мои · Ещё**.
- **Главная кнопка:** «кнопка-герой» — крупнейший элемент, закреплён (sticky)
  сверху на Ленте и Карте; механика **hold-to-send** с вибро и «отпустите для
  отправки».
- **Сценарий:** паника + спокойный путь раздельно. Красная кнопка = моментальная
  ТРЕВОГА (emergency) с авто-геолокацией, без формы; неэкстренное
  (правонарушение/внимание) — через отдельную спокойную форму.

## Архитектурные решения

### CSS-фреймворки не вводим
shadcn / theme-factory / Tailwind **не используем**. Они требуют Tailwind + Radix
— это крупная инвазия в проект, где сейчас ноль CSS и ноль `className`, плюс
лишний вес в офлайн-бандл PWA. Вместо этого — компактный ручной CSS-слой:
один файл токенов + по небольшому CSS-модулю на компонент. Решение обратимо: если
позже понадобится компонентная библиотека, это отдельная задача со своим спеком.

### Серверный API и доменную модель не трогаем
Вся работа — в `packages/web`. DTO из `@village/shared` и эндпоинты сервера
неизменны. Переиспользуем существующую логику: `outbox.enqueue`, `drainOutbox`,
`captureGeo` (с last-known fallback), `IncidentMap`, `Feed`, `Comments`,
`InstallPrompt`, `decodeAccessClaims`.

## Дизайн-система

Новый `src/theme.css`, импортируется в `main.tsx`. CSS-переменные:

- **Палитра (тёмная):** фон `#0b1220`, поверхность `#141d2e`, бордер `#1d2840`,
  текст `#e8edf6`, приглушённый текст `#5f7088`, сеть-онлайн `#3ddc84`.
- **Цвета уровней** (через существующий хук `data-level` на `<article>`):
  emergency `#ff3b3b`, offence `#ffb020`, attention `#3d8bff`.
- **Типографика:** тело — системный стек (быстро, читаемо, работает офлайн);
  дисплейный акцент для шапки/бейджей — один self-hosted вес конденсированного
  гротеска (без зависимости от сети). Базовый кегль 16–17px.
- **Размеры:** шаг отступов 4px; радиусы 12/18px; тап-цели ≥ 56px; контраст
  WCAG AA. Это требования доступности под пожилых и работу в перчатках.

## App shell и навигация

- **Хедер:** название села + индикатор сети (онлайн/офлайн).
- **Таб-бар (низ):** Лента · Карта · Мои · Ещё.
- **Роуты:** добавить `/map`, `/mine`, `/more` в `router/match.ts` и `routes.tsx`.
  Тип `Route` расширить новыми вариантами. На детали/форме — стрелка «назад» в
  хедере.
- **Sticky-герой** (`PanicButton` + ссылка «Сообщить о другом») показывается на
  Ленте и Карте, закреплён под хедером.

## Компонент `PanicButton` (ядро)

Поведение:
1. **Нажатие (pointerdown):** запустить `captureGeo()` в фоне; вибро-пульс
   (`navigator.vibrate`); запустить заполнение кольца (~1.5 с).
2. **Достигнут порог:** сменить подпись на «Отпустите для отправки»; вибро ещё
   раз.
3. **Отпустил после порога (pointerup):** собрать `NewIncidentInput` с
   `level: "emergency"` и захваченным гео (или last-known), без текста/медиа;
   `enqueue` + `drainOutbox`; перейти на экран инцидента, где можно дослать
   текст/фото.
4. **Отпустил до порога / увод пальца (pointercancel):** отмена, ничего не
   отправляется.

Защита от случайной отправки = порог удержания + «отпустите для отправки».
Доступность: `aria-label`, роль кнопки, видимое состояние прогресса.
Гео может не успеть — это допустимо: отправляем с тем, что есть (last-known
fallback уже реализован в `geo/capture.ts`).

«Дослать детали после отправки»: после паники открывается `/i/:id` с
формой комментария в фокусе (реюз `Comments`); фото — через существующий медиа-путь.

## Спокойный путь (неэкстренное)

Тихая ссылка «Сообщить о другом» рядом с героем ведёт на существующую форму
`CreateIncident` (роут `/new`), рестайл под тему. Выбор уровня offence/attention
(emergency тоже доступен, но основной путь для него — красная кнопка). Логика
формы не меняется, только оформление.

## Экраны (с переиспользованием)

| Роут | Экран | Делаем |
|---|---|---|
| `/` | Лента | `Feed` рестайл; sticky-герой; `IncidentCard` с цветными полосами уровня, статусом, временем, ⏳ pending |
| `/map` | Карта | реюз `IncidentMap` в display-режиме; пины по цвету уровня; тап → детали; sticky-герой |
| `/mine` | Мои | `Feed` с фильтром «мои» (id из `decodeAccessClaims`) |
| `/more` | Ещё | роль, статус сети, `InstallPrompt` (A2HS), тумблер push, выход, версия |
| `/i/:id` | Инцидент | `IncidentDetail` рестайл (Timeline, медиа-галерея, комментарии, действия командира) |
| `/new` | Создать | `CreateIncident` рестайл (спокойный путь) |
| `/register`, `/auth/callback` | Вход | рестайл под тему |

## Обратная связь и «живость»

- Индикатор сети в хедере; офлайн → «работаем офлайн, отправим когда появится сеть».
- ⏳ pending для элементов из outbox (уже есть).
- Явное подтверждение «Тревога отправлена» + переход к деталям.
- Тонкий slide-in новых карточек по SSE (источник уже есть — `useEventStream`).
- Дружелюбные пустые состояния («Пока тихо. Если что-то случилось — большая
  красная кнопка вверху»).
- Скелетоны вместо «Загрузка…».

## Вне объёма (YAGNI)

- CSS-фреймворки (Tailwind/shadcn/theme-factory).
- Изменения серверного API и доменной модели.
- Редактирование профиля, тонкие настройки уведомлений по типам сверх «Ещё».
- Кастомные карты/тайлы сверх того, что уже даёт `IncidentMap`.

## Тестирование

- **Сохранить все существующие `data-testid`** (`level-badge`, `status-badge`,
  `photo-input`, `geo-indicator`, `photo-preview` и пр.) — иначе придётся
  переписывать 101 web-тест. Рестайл не должен ломать селекторы.
- **Новые юниты:** `PanicButton` (порог удержания, отмена, вызов `vibrate`,
  вызов `enqueue` с level=emergency и гео), фильтр «Мои».
- **E2E (Playwright):** путь паники (hold → send → детали), спокойный путь через
  `/new`, навигация по табам, достижимость `/new` из UI.
- Зелёными остаются shared 4 + web (101 + новые) + server 65.

## Критерии готовности

- На `/new` есть вход из UI (закрыта функциональная дыра).
- Красная кнопка отправляет ТРЕВОГУ по hold-to-send с вибро и авто-гео.
- Все 4 вкладки работают; карта показывает инциденты пинами.
- Тёмная тема применена ко всем экранам; тап-цели ≥ 56px; контраст AA.
- Существующие тесты зелёные; добавлены новые юниты и E2E.
