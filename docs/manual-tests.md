# village-emrg — чеклист ручных проверок

То, что не покрывается unit/integration/E2E и требует реального устройства,
браузера или внешнего сервиса. Заполняется по мере реализации фронта (Plan 2)
и деплоя (Plan 3).

## Медиа: компрессия фото (Task 07)

Канвас и Web Worker не исполняются в jsdom — проверяется вручную в браузере:

- [ ] Загрузка фото с камеры/галереи (JPEG, PNG, HEIC где поддерживается).
- [ ] Результат — WebP, длинная сторона ≤ 1920 px.
- [ ] Размер после компрессии ≤ 1 МБ для типового фото с телефона.
- [ ] EXIF (включая GPS) отсутствует в результате (canvas отбрасывает метаданные).
- [ ] До 5 фото на инцидент; шестое не добавляется.
- [ ] Компрессия идёт в Worker — UI не подвисает на больших файлах.

## Service Worker: офлайн, кэш, push (Task 09)

SW целиком исполняется только в браузере (под E2E — Task 20, и вручную):

- [ ] Первая загрузка регистрирует SW; повторный визит работает офлайн (precache app shell).
- [ ] Навигация офлайн отдаёт закэшированный HTML (NetworkFirst → cache fallback).
- [ ] Фото инцидента после первого просмотра открывается офлайн (CacheFirst media).
- [ ] Создание инцидента офлайн ставится в Background Sync; при возврате сети
      POST `/incidents`, PUT в S3 и PATCH media реплеятся автоматически.
- [ ] Web-push показывает нотификацию из payload `{title, body, url}`.
- [ ] Клик по нотификации фокусирует уже открытую вкладку инцидента либо
      открывает deep-link.

## Web Push: подписка (Task 17)

Конвертация VAPID-ключа и формирование тела подписки покрыты unit-тестами;
реальная подписка требует браузера и (на iOS) установки A2HS:

- [ ] iOS Safari: подписка работает только после «Добавить на экран» (A2HS);
      запрос разрешения вызывается по жесту пользователя, не на старте.
- [ ] `Notification.requestPermission()` → `granted` → `pushManager.subscribe`
      с боевым `VITE_VAPID_PUBLIC_KEY`; подписка уходит на `PUT /push/subscription`.
- [ ] Отказ в разрешении не шлёт запрос на сервер и не ломает UI.
- [ ] Сервер реально доставляет уведомление подписанному устройству
      (проверяется вместе с боевыми VAPID-ключами на staging).
