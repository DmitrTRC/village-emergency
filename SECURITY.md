# Безопасность

## Сообщить об уязвимости

Не открывайте публичный issue для уязвимостей. Напишите приватно:

- через [GitHub Security Advisories](https://github.com/DmitrTRC/village-emergency/security/advisories/new);
- либо на 9984398@gmail.com с темой `[security] village-emrg`.

Ожидаемое время первичного ответа — 72 часа.

## Что особенно важно

Проект обрабатывает персональные данные жителей (адреса, ФИО, телефоны,
Telegram ID) в юрисдикции РФ (ФЗ-152). Приоритетные классы проблем:

- утечка PII (адреса/телефоны/ФИО) через API, логи или S3;
- обход авторизации (`policy.ts`: `canView` / `canComment` / `canAccept` / `canClose`);
- подделка Telegram-логина или сессий (reuse refresh-токена);
- доступ к чужим медиа в обход pre-signed URL;
- инъекции в текст инцидентов/комментариев (санитизация на бэке).

## Секреты

- Реальные секреты — только в `.env` (права `0600`), никогда в коде, тестах или коммитах.
- В репозитории — только `.env.example` с плейсхолдерами.
- Список домов и фамилий — вне репозитория (`data/houses.private.csv` в `.gitignore`).
